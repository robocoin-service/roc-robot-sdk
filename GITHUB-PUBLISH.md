# ROC Robot SDK GitHub Publish Guide

## Repository files

Create a public GitHub repository, for example:

```text
https://github.com/<your-account>/roc-robot-sdk
```

Put these files at the repository root:

```text
install.sh
roc-robot-tpm-sdk.sh
README.txt
README-TPM-LINUX.txt
```

## Platform frontend config

After the repository is created, replace these placeholders:

```text
https://github.com/robocoin-service/roc-robot-sdk.git
https://raw.githubusercontent.com/robocoin-service/roc-robot-sdk/main/install.sh
```

Recommended `.env` values for `roc-app`:

```env
VUE_APP_ROC_SDK_REPO_URL=https://github.com/<your-account>/roc-robot-sdk.git
VUE_APP_ROC_SDK_INSTALL_URL=https://raw.githubusercontent.com/<your-account>/roc-robot-sdk/main/install.sh
VUE_APP_AUTH_BASE_URL=https://your-deras-server.example.com
```

## User command after robot registration

The user only needs to SSH into the robot industrial PC and run the command generated by the platform page:

```bash
curl -fsSL https://raw.githubusercontent.com/<your-account>/roc-robot-sdk/main/install.sh | ROC_SDK_REPO_URL=https://github.com/<your-account>/roc-robot-sdk.git ROC_SERVER_URL=http://172.16.18.187:8090 bash -s -- <sdkBindingToken>
```

The binding token identifies the robot registration record. The SDK reads the TPM public key, asks the TPM private key to sign the binding payload, submits it to the server, then starts the DeRAS Agent.

## Security rule

The GitHub SDK can be public. The secret part is not inside the SDK. The real binding is controlled by:

- platform-generated one-time binding token
- TPM non-exportable private key
- server-side public-key verification
- token invalidation after successful binding
